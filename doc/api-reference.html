    tessla_server v0.0.1 API Reference

    Modules

    TesslaServer

    Entry Point for TesslaServer

    TesslaServer.Computation.Aggregation.EventCount

    Implements a Computation that emits a Signal holding the amount of Events happened on an Event Stream. To do so the state.operands list has to be initialized with one id representing the id of the Event Stream which events should be counted

    TesslaServer.Computation.Aggregation.EventMaximum

    Implements a Computation that emits the maximum value ever occured on an Event Stream or a default value if it’s bigger than all values occured to that point

    TesslaServer.Computation.Aggregation.EventMinimum

    Implements a Computation that emits the minimum value ever occured on an Event Stream or a default value if it’s smaller than all values occured to that point

    TesslaServer.Computation.Aggregation.Mrv

    Implements a Computation that emits a Signal holding the value of the latest event happened on the input. To do so the state.operands list has to be initialized with one id representing the id of the EventStream which values should be converted into a Signal and options has to hold the initial value under the key default

    TesslaServer.Computation.Aggregation.SignalMaximum

    Implements a Computation that emits the maximum value ever occured on an Signal Stream or a default value if it’s bigger than all values occured to that point

    TesslaServer.Computation.Aggregation.SignalMinimum

    Implements a Computation that emits the minimum value ever occured on an Signal Stream or a default value if it’s smaller than all values occured to that point

    TesslaServer.Computation.Aggregation.Sma

    Implements a Computation that emits an Event with the simple moving average of the last x input events every time a new Event is received. To do so the state.operands list has to be initialized with one id representing the id of the Event Stream which events should be the base for the computation and the options map has to hold a key count specifying the amount of Events which the average should be formed over

    TesslaServer.Computation.Aggregation.Sum

    Implements a Computation that emits a Signal holding the summed value of all events happened on the input EventStream. To do so the state.operands list has to be initialized with one id representing the id of the Event Stream which events’ values should be summed

    TesslaServer.Computation.Filter.ChangeOf

    Implements a Computation that emits Events with the value of a Signal whenever the Signal changes it’s value. To do so the state.operands list has to be initialized with one integer specifying the id of the Signal of which the changes should be emitted

    TesslaServer.Computation.Filter.Filter

    Implements a Computation that filters an event stream by the value of a boolean Signal

    TesslaServer.Computation.Filter.IfThen

    Implements a Computation that emits Events with the value of a Signal whenever an input Event is received

    TesslaServer.Computation.Filter.IfThenElse

    Implements a Computation that emits a Signal with the value of the second input Signal if the first boolean Signal is true or with the value of the third signal otherwise

    TesslaServer.Computation.Filter.Merge

    Implements a Computation that merges two event streams. The first specified stream takes presedence over the second, meaning if on both streams an event happens at the same time, the value of the first will be used

    TesslaServer.Computation.Filter.OccurAll

    Implements a Comptation that emits Events whenever both input stream are emitting an Event

    TesslaServer.Computation.Filter.OccurAny

    Implements a Computation that emits Events whenever one input stream is emitting an Event

    TesslaServer.Computation.Filter.Sample

    Implements a Computation that emits Events with the value of a Signal whenever an input Event is received

    TesslaServer.Computation.InputBuffer

    Functionality to buffer inputs to a computations and get the next events to compute

    TesslaServer.Computation.Lifted.Add

    Implements a Computation that adds two Signals

    TesslaServer.Computation.Lifted.And

    Implements a Computation that performs an and on two boolean signals

    TesslaServer.Computation.Lifted.Div

    Implements a Computation that divides two Signals

    TesslaServer.Computation.Lifted.Eq

    Implements a Computation that compares two integer Signals. if both are the same and false otherwise

    TesslaServer.Computation.Lifted.EventAbs

    Implements a Computation that computes the absolute value of an Event Stream

    TesslaServer.Computation.Lifted.EventNot

    Implements a Computation that computes the boolean not of a boolean Signal

    TesslaServer.Computation.Lifted.GenLifted

    Convenience wrapper for Lifted Computations working on two Signals and combine them somehow

    TesslaServer.Computation.Lifted.Geq

    Implements a Computation that compares two integer Signals and emits true if the first is greater or equal to the second and false otherwise

    TesslaServer.Computation.Lifted.Gt

    Implements a Computation that compares two integer Signals and emits true if the first is greater than the second and false otherwise

    TesslaServer.Computation.Lifted.Implies

    Implements a Computation that performs an implies on two boolean Signals

    TesslaServer.Computation.Lifted.Leq

    Implements a Computation that compares two integer Signals and returns true if the first is smaller or equal to the second and false otherwise

    TesslaServer.Computation.Lifted.Lt

    Implements a Computation that compares two integer Signals and returns true if the first is smaller than the second and false otherwise

    TesslaServer.Computation.Lifted.Max

    Implements a Computation that compares two integer signals and yields the value of the bigger one

    TesslaServer.Computation.Lifted.Min

    Implements a Computation that compares two integer Streams and returns the smaller one

    TesslaServer.Computation.Lifted.Mul

    Implements a Computation that multiplies two Signals

    TesslaServer.Computation.Lifted.Neg

    Implements a Computation that computes the negative value of a Signal

    TesslaServer.Computation.Lifted.Or

    Implements a Computation that performs an or on two boolean streams

    TesslaServer.Computation.Lifted.SignalAbs

    Implements a Computation that computes the absolute value of a Signal

    TesslaServer.Computation.Lifted.SignalNot

    Implements a Computation that computes the boolean not of a boolean Signal

    TesslaServer.Computation.Lifted.Sub

    Implements a Computation that substracts two event streams

    TesslaServer.Computation.Literal

    Represents a Literal

    TesslaServer.Computation.Monitors.Monitor

    Implements an Monitor based on multiple inputs, that are used in the LTL expression and a clock. state.options has to hold the id of the clock, the states, the initial state as current_state and the transitions

    TesslaServer.Computation.State

    Struct to represent the state of a Computation

    TesslaServer.Computation.Timing.DelayEventByCount

    Implements a Computation that delays input events until a later event occurs. The Number of events that need to occur must be specified under count in state.options. The Stream that should be delayed should be specified in state.operands as the only id in the list

    TesslaServer.Computation.Timing.DelayEventByTime

    Implements a Computation that delays the values of an Eventstream by the amount specified in options under the key amount in microseconds

    TesslaServer.Computation.Timing.DelaySignalByTime

    Implements a Computation that delays the values of a Signal by the amount specified in options under the key amount in microseconds. Only tested for positive values of amount. The output signal will hold the value specified under the key default in state.options as long as the first value is delayed

    TesslaServer.Computation.Timing.InPast

    Implements a Computation that emits a Signal holding the value true whenever an event happened on the input in a specified amount of time in the past. To do so the state.operands list has to be initialized with one id representing the id of the Event Stream which should be monitored and state.options has to hold the key amount specifying the timeslot where an event should have happened in past in microseconds

    TesslaServer.Computation.Timing.Timestamps

    Implements a Computation that emits an Event everytime an input Event is received and has its timestamp as it’s value. To do so the state.operands list has to be initialized with one id representing the id of the Event Stream which timestamps should be emitted

    TesslaServer.Event

    Represents an Event on a Stream. Note that Event here is meant in a broader sense than the Tessla one, as it can represent an event on an eventstream, a change of a signal or the progress of a stream

    TesslaServer.EventQueue

    Implements a central EventQueue to process all external Events in a strictly ordered way

    TesslaServer.GenComputation

    Base Module to work with Modules implementing the GenComputation behaviour. Note that Computations should implement specific GenServer calls, this cant be expressed right now by callbacks because the callbacks are allready defined in GenServer. Look at the methods below to see which messages are sent to a Computation

    TesslaServer.Output

    Used to log outputs. Has to be initialized at startup to hold the ids and names of streams that should be logged

    TesslaServer.Registry

    Helper functions for process registration

    TesslaServer.Source

    Entry Point for new Events from the monitored Programm

    TesslaServer.Source.FunctionCalls

    Implements a Source that emits events without a value whenever a specified Function is called. The function has to be specified as a String.t in options under the key function

    TesslaServer.Source.FunctionReturns

    Implements a Source that emits events without a value whenever a specified Function returns. The function has to be specified as a String.t in options under the key function

    TesslaServer.Source.InstructionExecutions

    Implements a Source that emits events without a value whenever a specified instruction is executed. The instruction has to be specified as a String.t in options under the key instruction

    TesslaServer.Source.VariableValues

    Implements a Source that emits a Signal holding the most recent value of a variable. The variable has to be specified as a String.t in options under the key variable

    TesslaServer.SpecProcessor

    Entry point for processing a Tessla Spec generated by the Tessla Compiler

    TesslaServer.SpecProcessor.GraphBuilder

    Takes a Tessla spec and generates a DAG of TesslaServer.GenComputation modules which represents the spec
