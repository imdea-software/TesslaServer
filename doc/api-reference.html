    tessla_server v0.0.1 API Reference

    • Modules

    Modules

    TesslaServer

    Entry Point for TesslaServer

    TesslaServer.Event

    Contains the struct to represents Events which are modified by TesslaServer.Node implementations

    TesslaServer.EventQueue

    Implements a central EventQueue to process all external Events in a strictly ordered way

    TesslaServer.EventStream

    Represents a Stream, generated either from external Events or by computation through a Node Note that progressed_to can be bigger than the timestamp of the latest Event: If all Inputs of the Node generating this Stream have advanced to at least t but no new Event is generated (e.g. because the Node is a delay with a higher value) the Stream will be progressed to t but no Event with a timestamp t is present

    TesslaServer.Node

    Base Module to work with Modules implementing the Node behaviour. Note that nodes should implement specific GenServer calls, this cant be expressed right now by callbacks because the callbacks are allready defined in GenServer. Look at the methods below to see which messages are sent to a Node

    TesslaServer.Node.Aggregation.EventCount

    Implements a Node that emits a Signal holding the amount of Events happened on an Event Stream. To do so the state.operands list has to be initialized with one id representing the id of the Event Stream which events should be counted

    TesslaServer.Node.Aggregation.EventMaximum

    Implements a Node that emits the maximum value ever occured on an Event Stream or a default value if it’s bigger than all values occured to that point

    TesslaServer.Node.Aggregation.EventMinimum

    Implements a Node that emits the minimum value ever occured on an Event Stream or a default value if it’s smaller than all values occured to that point

    TesslaServer.Node.Aggregation.Mrv

    Implements a Node that emits a Signal holding the value of the latest event happened on the input. To do so the state.operands list has to be initialized with one id representing the id of the EventStream which values should be converted into a Signal

    TesslaServer.Node.Aggregation.SignalMaximum

    Implements a Node that emits the maximum value ever occured on an Signal Stream or a default value if it’s bigger than all values occured to that point

    TesslaServer.Node.Aggregation.SignalMinimum

    Implements a Node that emits the minimum value ever occured on an Signal Stream or a default value if it’s smaller than all values occured to that point

    TesslaServer.Node.Aggregation.Sma

    Implements a Node that emits an Event with the simple moving average of the last x input events every time a new Event is received. To do so the state.operands list has to be initialized with one id representing the id of the Event Stream which events should be the base for the computation and the options map has to hold a key count specifying the amount of Events which the average should be formed over

    TesslaServer.Node.Aggregation.Sum

    Implements a Node that emits a Signal holding the summed value of all events happened on the input EventStream. To do so the state.operands list has to be initialized with one id representing the id of the Event Stream which events’ values should be summed

    TesslaServer.Node.Filter.ChangeOf

    Implements a Node that emits Events with the value of a Signal whenever the Signal changes it’s value. To do so the state.operands list has to be initialized with one integer specifying the id of the Signal of which the changes should be emitted

    TesslaServer.Node.Filter.Filter

    Implements a Node that filters an event stream by the value of a boolean Signal

    TesslaServer.Node.Filter.IfThen

    Implements a Node that emits Events with the value of a Signal whenever an input Event is received

    TesslaServer.Node.Filter.IfThenElse

    Implements a Node that emits a Signal with the value of the second input Signal if the first boolean Signal is true or with the value of the third signal otherwise

    TesslaServer.Node.Filter.Merge

    Implements a Node that merges two event streams. The first specified stream takes presedence over the second, meaning if on both streams an event happens at the same time, the value of the first will be used

    TesslaServer.Node.Filter.OccurAll

    Implements a Node that emits Events whenever both input stream are emitting an Event

    TesslaServer.Node.Filter.OccurAny

    Implements a Node that emits Events whenever one input stream is emitting an Event

    TesslaServer.Node.Filter.Sample

    Implements a Node that emits Events with the value of a Signal whenever an input Event is received

    TesslaServer.Node.History

    Contains the Structure and Methods to work with the History of a TesslaServer.Node

    TesslaServer.Node.Lifted.Add

    Implements a Node that adds two event streams

    TesslaServer.Node.Lifted.And

    Implements a Node that performs an and on two boolean streams

    TesslaServer.Node.Lifted.Div

    Implements a Node that divides two event streams

    TesslaServer.Node.Lifted.Eq

    Implements a Node that compares two integer Streams and returns true if both are the same and false otherwise

    TesslaServer.Node.Lifted.EventAbs

    Implements a Node that computes the absolute value of a stream

    TesslaServer.Node.Lifted.EventNot

    Implements a Node that computes the boolean not of a boolean stream

    TesslaServer.Node.Lifted.Geq

    Implements a Node that compares two integer Streams and returns true if the first is greater or equal to the second and false otherwise

    TesslaServer.Node.Lifted.Gt

    Implements a Node that compares two integer Streams and returns true if the first is greater than the second and false otherwise

    TesslaServer.Node.Lifted.Implies

    Implements a Node that performs an implies on two boolean streams

    TesslaServer.Node.Lifted.Leq

    Implements a Node that compares two integer Streams and returns true if the first is smaller or equal to the second and false otherwise

    TesslaServer.Node.Lifted.Lt

    Implements a Node that compares two integer Streams and returns true if the first is smaller than the second and false otherwise

    TesslaServer.Node.Lifted.Max

    Implements a Node that compares two integer Streams and returns the bigger one

    TesslaServer.Node.Lifted.Min

    Implements a Node that compares two integer Streams and returns the smaller one

    TesslaServer.Node.Lifted.Mul

    Implements a Node that multiplies two event streams

    TesslaServer.Node.Lifted.Neg

    Implements a Node that computes the negative value of a stream

    TesslaServer.Node.Lifted.Or

    Implements a Node that performs an or on two boolean streams

    TesslaServer.Node.Lifted.SignalAbs

    Implements a Node that computes the absolute value of a stream

    TesslaServer.Node.Lifted.SignalNot

    Implements a Node that computes the boolean not of a boolean stream

    TesslaServer.Node.Lifted.Sub

    Implements a Node that substracts two event streams

    TesslaServer.Node.Literal

    Represents a Literal

    TesslaServer.Node.Monitors.Monitor

    Implements an Monitor based on multiple inputs, that are used in the LTL expression and a clock. state.options has to hold the id of the clock, the states, the initial state as current_state and the transitions

    TesslaServer.Node.State

    Struct to represent the state of a Node

    TesslaServer.Node.Timing.DelayEventByCount

    Implements a Node that delays input events until a later event occurs. The Number of events that need to occur must be specified under count in state.options. The Stream that should be delayed should be specified in state.operands as the only id in the list

    TesslaServer.Node.Timing.DelayEventByTime

    Implements a Node that delays the values of an EventStream by the amount specified in options under the key amount in microseconds

    TesslaServer.Node.Timing.DelaySignalByTime

    Implements a Node that delays the values of a signal Stream by the amount specified in options under the key amount in microseconds. Only tested for positive values of amount. The output signal will hold the value specified under the key default in state.options as long as the first value is delayed

    TesslaServer.Node.Timing.InPast

    Implements a Node that emits a Signal holding the value true whenever an event happened on the input in a specified amount of time in the past. To do so the state.operands list has to be initialized with one id representing the id of the Event Stream which should be monitored and state.options has to hold the key amount specifying the timeslot where an event should have happened in past in microseconds

    TesslaServer.Node.Timing.Timestamps

    Implements a Node that emits an Event everytime an input Event is received and has it’s timestamp as it’s value. To do so the state.operands list has to be initialized with one id representing the id of the Event Stream which timestamps should be emitted

    TesslaServer.Output

    Used to log outputs. Has to be initialized at startup to hold the ids and names of nodes that should be logged

    TesslaServer.Registry

    Helper functions for node registration

    TesslaServer.SimpleNode

    Basic implementation of the Node behaviour that can be used by concrete Nodes to bootstrap

    TesslaServer.Source

    Entry Point for new Events from the monitored Programm

    TesslaServer.Source.FunctionCalls

    Implements a Source that emits events without a value whenever a specified Function is called. The function has to be specified as a String.t in options under the key function

    TesslaServer.Source.FunctionReturns

    Implements a Source that emits events without a value whenever a specified Function returns. The function has to be specified as a String.t in options under the key function

    TesslaServer.Source.InstructionExecutions

    Implements a Source that emits events without a value whenever a specified instruction is executed. The instruction has to be specified as a String.t in options under the key instruction

    TesslaServer.Source.VariableValues

    Implements a Source that emits a Signal holding the most recent value of a variable. The variable has to be specified as a String.t in options under the key variable

    TesslaServer.SpecProcessor

    Entry point for processing a Tessla Spec generated by the Tessla Compiler

    TesslaServer.SpecProcessor.GraphBuilder

    Takes a Tessla spec and generates a DAG of TesslaServer.Node modules which represents the spec

    TesslaServer.SpecProcessor.Lexer

    Lexes the cleaned and structured tessla spec

    TesslaServer.SpecProcessor.Parser

    Invokes the parser on the lexed input

    TesslaServer.SpecProcessor.Preprocessor

    Preprocesses a Tessla Spec
